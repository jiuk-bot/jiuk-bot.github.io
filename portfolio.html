CASE 1 — 결제 전환 구조 개선으로 매출 16배+

셀프스토리지 · 무인 게임룸 서비스 · 박스풀 · 2024.07 ~ 2025.06
① 문제

GA4상 예약 페이지 조회 수 1,698회였지만 결제는 0건. 예약이 '고객센터 전화'로만 가능해, 바로 결제하려던 수요가 이탈 — 관심이 아닌 결제 구조의 병목이라 판단했습니다.

② 가설

경쟁사 조사 결과 게임룸 특성상 며칠 전 사전 예약보다 '당일 예약'이 많다는 걸 확인 → '결제 장벽 낮추기'가 핵심이라 판단했습니다.

③ 해결
  • 고객센터 전화 예약 → 즉시 예약·결제 시스템 도입으로 전환 병목 제거
  • 취미공간 전용 LP 신설 + 메인 노출로 전환 경로 단축
④ 결과
구분페이지 조회결제 전환
도입 전1,698회0건
도입 후738회16건

결제 전환 0 → 16건 · 매출 16배+ (1,653%).

핵심 정리: 신규 서비스의 전환 병목을 직접 진단·설계해 매출(0 to 1)로 연결했습니다.

CASE 2 — GA4 구매전환 퍼널 설계 → 신규고객 +63%

셀프스토리지 · 짐보관 서비스 · 박스풀 · 2024.07 ~ 2025.06
① 문제

전환 경로 데이터의 부재. 고객이 결제까지 가는 경로·소요 시간·채널별 기여도를 알 수 없어 리타게팅 타이밍의 판단 근거와 퍼널 구조가 모두 없었습니다.

② 가설 (데이터로 발견)

GA4로 고객 여정 분석 → 결제까지 평균 3.89일, 채널별 기여 단계 상이 확인 → '즉시 결제가 아닌 고관여 상품'으로 판단. 단발 노출이 아닌 결제 여정 재구성 + 타이밍 리타게팅이 필요했습니다.

③ 해결 — 4단계 퍼널 + 타이밍 리타게팅

채널별 역할(① 검색=첫 유입 → ② 블로그=정보·신뢰 → ③ Meta=재접촉 → ④ 결제)과 3.89일을 확인해 그 순서·타이밍에 맞춰 설계했습니다.

  • ① 검색 — 검색광고로 포털 첫 유입 → 홈페이지 도달
  • ② 블로그 — 자사·경쟁사 홈페이지에 리뷰가 없어 소비자가 블로그에서 정보를 얻는 점에 착안 → 후기·차별점 리뷰 발행으로 신뢰 지점 마련
  • ③ Meta — 유입자 리타게팅으로 3.89일 타이밍 리마인드, 결제 직전 재접촉
  • ④ 결제 — 리타게팅 클릭 시 예약 페이지 직접 랜딩 + 프로모션으로 이탈 최소화
  • 타겟 정교화 — 자사·경쟁사 홈페이지 방문자 + 유사타겟 2% + 픽셀 세분화로 실구매 가능성 높은 유저에 집중

CASE 3 — KB국민은행 금융 캠페인 (앱실행 단가 -41% · -29%)

블루오렌지커뮤니케이션즈 (광고주 KB국민은행) · 5채널 · 집행 약 7.6억 · 2025.09 ~ 2025.12
① 스타틴즈 캠페인 (10대·5채널)

메타·유튜브·토스·틱톡·콴다 5채널 균등분배로 시작 → 효율 기반 능동 재배분. KPI는 앱 실행(앱전환) ↑ · 앱실행 단가 ↓.

  • 효율 기반 예산 재배분 — 유튜브 기존고객 리타겟이 모수가 적어 CPV가 관심사 타겟의 약 3배로 비효율 → 예산을 관심사 타겟으로 이관 (관심사 CPV -38%)
  • 코어 타겟 집중 — 틱톡 노출 57%·조회 69%가 45세 이상, 정작 10대(13–17)는 노출 5% → 연령 13–44 축소 + 65+ 디타겟 + 학부모(교육·홈스쿨링) 관심사 추가로 예산이 코어 타겟(10대·학부모)에 닿도록 재조정

→ 앱실행 단가 -41% 개선 (운영 초기 대비).

② 패밀리뱅킹 캠페인 (30~50대·3채널)

KB 자사 고객 데이터(1st-party) 기반 유튜브·토스·인스타 3채널 운영.

  • '신규 획득'보다 '기존 고객 활성화' — 자녀 관련 세그먼트가 무관 세그먼트보다 앱실행률 약 5배 → 신규 획득보다 자녀 있는 기존 고객 활성화가 효율적이라 판단, KB 1st-party로 자녀 적금·모임통장 세그먼트 집중 제안
  • 매체 효율 재배분(인스타→토스) · 채널 역할 분담(유튜브=조회 / 토스=앱전환)

→ 앱실행 단가 -29% 개선 (운영 초기 대비).

핵심 정리: 균등분배가 아니라 지표로 저효율 → 고효율 재배분하고, 신규 획득보다 1st-party 세그먼트 활성화로 앱실행 단가를 낮췄습니다.

CASE 4 — 다채널 실시간 데이터 분석 대시보드 직접 구축

치과 임플란트 광고 (구글애즈·메타·틱톡) · 글로벌랩스 · 2026.01 ~ 현재
① 문제

뎁스 분석에 필요한 로우데이터 가공에 많은 시간이 투여돼, 정작 분석할 시간이 부족해지는 상황이었습니다.

② 해결 — 가공 자동화 + 지표 직접 설계
  • 구글애즈·메타·틱톡 3채널을 API로 연동 → 원하는 데이터를 실시간 자동 가공·분석
  • 히스토리 기록·조회 · A/B 테스트 스케줄 알림 · 유튜브 노출 채널 변화 추이까지 한 화면에
  • 중복DB율 · 실전환 CPA · 캠페인 수명 코호트 · 경쟁사 소재/LP 스크래핑 등 기성 지표로 안 보이는 분석 지표를 직접 설계·자동화
③ 결과

데이터 가공 자동화로 분석에 쓰는 시간이 늘고, 그만큼 뎁스 있는 로우데이터까지 분석 가능. 대시보드 활용으로 대행 제로베이스에서 ROAS 336% 달성(→ CASE 6).

핵심 정리: 필요한 분석을 직접 도구화 — 수작업에 뺏기던 시간을 분석에 투자하고, 필요 지표를 직접 설계해 빠르고 깊은 분석으로 다채널 ROAS를 개선했습니다.
운영 의사결정 · 5 사례

CASE 5 — 실시간 대시보드 활용 운영·분석 사례

치과 임플란트 (구글 디멘드젠 · 메타 · 틱톡) · 글로벌랩스
① 노출 유튜브 채널을 직접 추적 (구글 디멘드젠)

CPA만으론 타겟 풀 소진·신규 풀 진입을 알 수 없어, 광고 노출 유튜브 채널을 대시보드에 시각화·자동화. 일별 채널 교체율 32~53%를 정상 베이스라인으로 설정하고, 세그먼트 변경이 실제로 노출 채널을 바꿨는지 추적해 '신규 풀 진입으로 CPA 개선'을 데이터로 검증했습니다.

② 관심사 캠페인 신규 운영 → CPA 약 26% 개선 (구글 디멘드젠)

연령만 타겟(Mass)으로 운영하며 실제 전환 고객의 관심사 5개를 추출해 별도 캠페인을 운영. 동기간 비교 시 관심사 캠페인 CPA 약 26% 개선, 캠페인간 중복DB로 자가잠식을 검증한 결과 관심사 고객의 약 79%가 신규 연락처 = 신규 고객으로 풀 확장.

③ 새벽 일괄 차단 대신 캠페인별 차등 선별 차단 (구글 디멘드젠)

시간대별 CPA 분석 결과 캠페인마다 새벽(0~5시) CPA가 그 외 대비 +45~58% 비효율. 비효율 새벽 구간만 선별 차단 → 일괄 차단이 낮(-14%)·저녁(-19%) 전환을 새벽으로 분산시키는 카니발리제이션을 데이터로 확인·회피하고, 새벽 낭비 예산을 효율 시간대로 재배분.

④ 고객 DB 유사타겟 4% vs 6% 확장 A/B (메타)

"좁게(4%)면 질↑ 볼륨↓, 넓게(6%)면 볼륨↑ 질↓" 가설을 검증. 결과 6% 확장이 4% 대비 CPA -22%인데 CVR은 거의 동등 → 타겟 모수 확장이 효율 우위임을 확인.

⑤ 고객 DB 유사세그먼트 vs 광범위 (틱톡)

광범위(Mass)만으론 질 한계 → 고객 DB 유사세그먼트를 별도 운영·비교. 유사세그먼트가 광범위 대비 CVR 약 1.9배 · CPA -11%.

핵심 정리: 표면 숫자가 아니라 지표를 분해해 원인을 짚고, 그 근거로 운영을 결정했습니다.
★ 대표 성과 · ROAS 3배

CASE 9 — 창업설명회: 0 to 1 신규 사업 직접 런칭

셀프스토리지 프랜차이즈 가맹(B2B) · 박스풀 · 2024.07 ~ 2025.06
① 문제

가맹 모집인데 전화 상담만으론 계약 전환이 약했고, 창업 의향이 높은 고객 위주의 질 높은 상담이 필요하다 판단했습니다.

② 가설

창업 같은 큰 결정은 전화로 안 움직임 → 제안서로 신뢰를 주고 직접 실물을 보게 하면 인원이 적어도 전환↑.
